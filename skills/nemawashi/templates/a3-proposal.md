# A3 Proposal Template

When proposing a major refactor or architectural change, the agent MUST format the findings using this exact schema to secure consensus before execution.

```markdown
# A3 Proposal: [Refactor/Change Name]

## 1. Background
[Context of the proposed change and why it is being considered.]

## 2. Current Condition
- **Gemba Facts:** [What is the factual state of the code now?]
- **Impacted Areas:** [List of files, modules, or tests impacted]

## 3. Goal
[The desired outcome or target state.]

## 4. Analysis (5-Whys)
[Why the change is necessary; root cause analysis of the current pain points.]

## 5. Countermeasures
[The specific technical implementation steps intended to resolve the issue.]

## 6. Execution Plan
- [ ] Step 1: [Action]
- [ ] Step 2: [Action]

## 7. Follow-up
- **Verification:** [How we will empirically verify success (Genchi Genbutsu)]
```
